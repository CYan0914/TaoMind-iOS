## 🔴 必须修复（否则会被拒绝或无法通过审核）

### 1. 隐私政策 & 服务条款 URL 需要可访问
- **问题：** 代码中 `https://taomind.app/privacy` 和 `https://taomind.app/terms`——域名未注册/未解析
- **解决方案：** 二选一
  - **方案 A（推荐）：** 用 GitHub Pages 免费托管，代码中把 URL 改成 GitHub Pages 地址
  - **方案 B：** 注册 `taomind.app` 域名（约 ¥30-50/年）→ 在 SettingsView.swift 更新 URL
  - **方案 C：** 直接用隐私政策生成器的托管链接
- **涉及文件：** `Views/SettingsView.swift` 第 87-88 行
  ```swift
  Link("Privacy Policy", destination: URL(string: "https://taomind.app/privacy")!)
  Link("Terms of Service", destination: URL(string: "https://taomind.app/terms")!)
  ```

### 2. 需要 App Icon
- **问题：** `ASSETCATALOG_COMPILER_APPICON_NAME: ""` — 空图标
- **解决方案：** 在 Mac 上：
  1. 打开 Xcode → `Resources/Assets.xcassets`
  2. 新建 AppIcon（Asset Catalog 里右键 → App Icon）
  3. 拖入 1024×1024 PNG 图标
  4. 编辑 `project.yml` 删除 `ASSETCATALOG_COMPILER_APPICON_NAME: ""`（第 29 行）
  5. 提交代码，重新触发 CI 构建上传

### 3. NSAllowsArbitraryLoads — 需要说明理由
- **问题：** Info.plist 中设置为 `true`，允许所有 HTTP 连接。App Store 审核需要理由
- **解决方案：** 二选一
  - **最佳方案：** 改为仅允许你的 API 域名（推荐）：
    ```xml
    <key>NSAppTransportSecurity</key>
    <dict>
      <key>NSExceptionDomains</key>
      <dict>
        <key>observant-prosperity-production-92d3.up.railway.app</key>
        <dict>
          <key>NSExceptionAllowsInsecureHTTPLoads</key>
          <true/>
        </dict>
      </dict>
    </dict>
    ```
  - **或者：** 在审核备注中说明开发/测试需要，提交前临时关闭提交审核后恢复

### 4. 代码中的硬编码 Token/API Key
- **问题：** 这个需要在代码中检查是否有硬编码的 API key 或 token。让用户确认。

## 🟡 建议修复（会影响用户体验或可能出现问题）

### 1. 收藏按钮无后端存储
- **问题：** WisdomResultView 里的 heart/favorite 按钮只切换本地状态，不保存到服务器
- **影响：** 用户收藏后重启应用，收藏状态丢失
- **建议：** 上架前至少注释掉这个按钮，或加一行提示"收藏功能即将上线"

### 2. "Save Notes" 按钮无效
- **问题：** JournalEntryDetailView 的 "Save Notes" 按钮 action 体为空
- **影响：** 用户可以写笔记但点保存没反应，造成困惑
- **建议：** 要么实现保存逻辑，要么暂时隐藏该功能

### 3. "Restore Purchases" 死链接
- **问题：** 链接到 `"#"`，点击后无反应
- **影响：** 用户体验不佳
- **建议：** 在 IAP 实现前暂时移除或注释掉

### 4. "Upgrade to Premium" 按钮无响应
- **问题：** `Button(action: { /* open IAP */ })` —— 无实际操作
- **影响：** 免费版用户可以正常使用，但如果看到升级按钮点了没反应会有困惑
- **建议：** 上架前将按钮改为即将推出样式，或添加 Alert："Coming Soon"

### 5. 每日经文自动推送
- **问题：** DailyVerseCard 在应用启动时加载一次，但没有主动推送
- **影响：** 用户预期"每日"→但需要每天打开 App 才能看到新的
- **建议：** 非必须，后续可考虑本地通知

- 后端目前可能没有 DELETE endpoint

### 2. 缺少空状态展示
- Journal 为空时显示空白列表，没有引导文案

### 3. 请求失败时的用户提示
- 目前网络请求失败时，部分场景没有给用户弹错误提示

### 4. 代码中 PRO 限制未实施
- `isPro` 标志没有在代码中任何地方使用来限制功能
- 所有功能在免费版下完全可用
